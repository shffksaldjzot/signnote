# 시작할 때 `/bkit` 실행해서 도움받기

# 절대 기준 (반드시 준수)

## 워킹로그 업데이트
- 배포 직전에 반드시 `WORKLOG.md`를 업데이트한다
- 해당 세션에서 작업한 내용을 빠짐없이 기록

## 숫자 필드 콤마 포맷
- 모든 숫자 입력 필드에는 단위 구분 콤마(천 단위)를 반드시 적용한다 (예: 500000 → 500,000)
- 표시할 때도 콤마 포함, 입력할 때도 실시간 콤마 자동 삽입
- **예외**: 전화번호, 사업자등록번호, 수수료율(%)은 콤마 없이 그대로 입력
- 서버에 보낼 때는 콤마 제거 후 숫자만 전송

## 디자인 가이드라인 (2차 디자인)
- **폰트**: Inter (전체 앱 통일)
- **역할별 메인컬러**:
  - 고객(CUSTOMER): `#2D6EFF` (파란색)
  - 주관사(ORGANIZER): `#FF6A00` (주황색)
  - 협력업체(VENDOR): `#000000` (검정색)
- **하단 네비바 아이콘**: `assets/icons/` 폴더의 PNG 아이콘 사용 (디자인 제공)
- **주관사 하단바**: 홈 / 마이페이지 (2탭)
- **고객 하단바**: 홈 / 장바구니 / 계약함 / 마이페이지 (4탭)
- **협력업체 하단바**: 홈 / 마이페이지 (2탭)
- **행사 상세 상단 탭**: 품목관리 / 고객관리 / 계약함 / 알림 (4탭)
- **참여코드**: 행사당 고객용 6자리 + 협력업체용 6자리 분리 생성
