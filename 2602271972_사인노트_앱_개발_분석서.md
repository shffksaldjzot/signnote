# 사인노트(SignNote) 앱 개발 종합 분석서

---

## 1. 디자인 파일 분석 결과

업로드해주신 디자인을 모두 확인했습니다. 총 **19개 화면**으로 구성되어 있으며, 3가지 사용자 유형별로 잘 정리되어 있습니다.

| 구분 | 화면 수 | 주요 화면 |
|------|---------|----------|
| **고객용 (사용자페이지)** | 8개 | 코드 입장 → 타입 선택 → 행사 목록 → 행사 상세 → 품목 상세 → 장바구니 → 계약함 |
| **협력업체용 (업체페이지)** | 5개 | 코드 입장 → 행사 목록 → 품목 상세 → 계약함(집계 + 계약 건 관리) |
| **주관사용 (주관사페이지)** | 6개 | 행사 목록 → 행사 추가 → 행사 관리 → 품목 상세 관리 |

### 비즈니스 플로우 요약

```
주관사(행사 생성/관리)
    ↓ 참여 코드 발급
협력업체(품목 등록/계약 관리) ←→ 고객(품목 선택 → 장바구니 → 계약금 결제)
    ↓
계약금 결제 → PG 정산 → 업체별 금액 분배
```

## 2. 클로드가 디자인 이미지대로 앱을 만들 수 있는가?

### ✅ 가능한 것

- **UI/프론트엔드 구현**: 디자인 이미지를 보고 동일한 레이아웃, 색상, 컴포넌트를 코드로 구현하는 것은 가능합니다. 제공해주신 디자인은 깔끔한 카드형 UI, 바텀 네비게이션, 모달 팝업 등 표준적인 모바일 앱 패턴으로 되어 있어 구현 난이도가 적절합니다.
- **프로토타입/데모 앱**: 한 번의 대화에서 동작하는 프로토타입 수준의 UI를 만들 수 있습니다.

### ⚠️ 한계가 있는 것

- **픽셀 퍼펙트 구현**: 이미지와 100% 완벽하게 동일한 픽셀 단위 구현은 어렵습니다. 약 90~95% 수준의 유사도로 구현 가능합니다.
- **전체 앱을 한 번에 완성**: 19개 화면 + 백엔드 + DB + 결제 + 알림까지 한 번의 대화로 완성하기는 어렵습니다. 화면 단위로 나누어 점진적으로 구현하는 것이 현실적입니다.
- **실제 배포 수준의 보안**: 보안 감사, 취약점 테스트 등은 별도 전문 업체의 검토가 필요합니다.

### ❌ 클로드가 직접 할 수 없는 것

- 앱스토어/플레이스토어 배포 (개발자 계정 및 서명 필요)
- PG사 실제 계약 및 가맹점 심사
- 카카오톡 알림톡 발송 채널 등록 (사업자 인증 필요)
- SSL 인증서 발급 및 서버 호스팅

## 3. 추천 개발 언어 및 기술 스택

사인노트의 요구사항(PC 관리자 + 모바일 고객/업체 + 앱 패키징)을 고려할 때, 아래 두 가지 방식을 추천합니다.

### 방식 A: Flutter + Node.js (⭐ 최우선 추천)

| 영역 | 기술 | 이유 |
|------|------|------|
| **모바일 앱 (고객/업체)** | Flutter (Dart) | 하나의 코드로 iOS + Android 동시 개발. 네이티브 성능. 앱 패키징 용이 |
| **관리자 웹 (PC)** | Flutter Web 또는 React | 관리자는 PC 브라우저 접속이므로 웹 대시보드 |
| **백엔드 서버** | Node.js (NestJS) | PG사 연동 SDK 지원 풍부, 카카오 알림톡 API 연동 용이 |
| **데이터베이스** | PostgreSQL + Redis | 트랜잭션 안전성(결제), 세션 캐싱 |
| **앱 푸시** | Firebase Cloud Messaging (FCM) | iOS/Android 통합 푸시 알림 |

### 방식 B: React Native + Node.js (차선)

| 영역 | 기술 | 이유 |
|------|------|------|
| **모바일 앱** | React Native | 웹 개발자 친화적(JavaScript 기반) |
| **관리자 웹** | React (Next.js) | 코드 재사용 가능 |
| **백엔드** | Node.js (NestJS) | 동일 |

### 왜 Flutter를 최우선 추천하는가

1. **단일 코드베이스**: iOS, Android, Web 모두 하나의 코드로 개발 가능
2. **네이티브 성능**: React Native보다 렌더링 성능이 우수
3. **앱 패키징**: 네이티브 APK/IPA 빌드가 안정적
4. **보안**: Dart 언어의 AOT 컴파일로 코드 역공학이 상대적으로 어려움
5. **UI 구현력**: 디자인에 있는 모달, 바텀시트, 카드 UI 등을 쉽게 구현 가능

## 4. 보안 관련 핵심 사항

사인노트는 **계약과 결제가 핵심**이므로 보안이 매우 중요합니다.

### 필수 보안 요소

| 항목 | 설명 |
|------|------|
| **SSL/TLS (HTTPS)** | 모든 통신 암호화 필수 |
| **JWT + Refresh Token** | 사용자 인증 토큰 관리 |
| **API Rate Limiting** | 무차별 대입 공격 방지 |
| **데이터 암호화** | 개인정보(이름, 연락처) AES-256 암호화 저장 |
| **PG사 보안 규격 준수** | PCI-DSS 준수, 카드 정보 비보유(PG사에 위임) |
| **역할 기반 접근 제어 (RBAC)** | 관리자/주관사/업체/고객별 권한 분리 |
| **감사 로그** | 계약/결제/취소 등 모든 중요 행위 로그 기록 |
| **SQL Injection 방지** | ORM(Prisma, TypeORM) 사용 |
| **앱 위변조 방지** | 코드 난독화, 루팅/탈옥 탐지 |

### 개인정보 보호법 준수

- 개인정보 처리방침 작성 및 공개
- 개인정보 수집·이용 동의 절차 구현
- 개인정보 보호책임자 지정
- 데이터 보관 기간 설정 및 파기 절차

## 5. PG사 결제 및 정산 분배

### 핵심 질문: "500만원을 결제받고 5개 업체에 자동으로 나눠줄 수 있나?"

### ✅ 가능합니다 — "지급대행" 서비스 활용

KG이니시스의 **지급대행 서비스**가 정확히 이 요구사항에 맞습니다. 오픈마켓이나 플랫폼 사업자가 하위 판매자들에게 결제대금을 자동으로 분배 정산할 수 있도록 해주는 서비스입니다.

#### 지급대행 서비스 제공 PG사

| PG사 | 서비스명 | 특징 |
|------|---------|------|
| **KG이니시스** | 지급대행 서비스 | 하위 판매자 계좌정보 및 지급데이터를 기반으로 자금 정산을 대행 |
| **토스페이먼츠** | 서브몰 정산 | 플랫폼 내 하위 상점별 자동 정산 지원 |
| **NHN KCP** | 지급대행 | 대표가맹점 방식으로 하위 가맹점 정산 |

#### 작동 방식 (사인노트 기준)

```
1. 고객이 500만원 결제 (줄눈 140만 + 나노코팅 70만 + ...)
                    ↓
2. PG사가 대금 수령 (수수료 차감)
                    ↓
3. 사인노트 서버가 PG사에 정산 분배 데이터 전송
   - A업체(줄눈): 140만원 → A업체 계좌
   - B업체(나노코팅): 70만원 → B업체 계좌
   - C업체(...): ... → C업체 계좌
   - 사인노트 수수료: X만원 → 사인노트 계좌
                    ↓
4. PG사가 각 업체 계좌로 자동 입금
```

#### ⚠️ 주의사항

- **전자지급결제대행업(PG업) 등록 필요 여부**: 사인노트가 직접 정산에 관여하는 구조라면 전자금융거래법상 PG업 등록이 필요할 수 있습니다. 다만 정산업무를 외부 PG사에 위탁하면 등록 의무가 면제됩니다.
- **보증보험 가입**: PG사 계약 시 필수
- **수수료**: 신용카드 기준 약 3.2~3.5% (영세/중소 사업자는 우대 수수료 적용)

## 6. 세금계산서 자동 발행

### ✅ 완전 자동화 가능 — 팝빌 또는 바로빌 API 활용

| 서비스 | 특징 | 가격 |
|--------|------|------|
| **팝빌 (Popbill)** | 국세청 표준인증, Bulk API 지원 (1분에 1만건), Webhook 지원, 14개+ 개발언어 SDK | 건당 과금 |
| **바로빌 (BaroBill)** | 정발행/역발행/위수탁발행 모두 지원, 재판매 구조 가능 | 건당 과금 |
| **홈택스빌** | 6개월 무료, 대량 발행 지원 | 건당 과금 |

#### 자동화 시나리오 (사인노트 기준)

```
1. 고객이 계약금 결제 완료
                    ↓
2. 사인노트 서버가 자동으로 세금계산서 데이터 생성
   - 공급자: 해당 협력업체
   - 공급받는자: 고객 (또는 사인노트)
   - 금액: 해당 품목 금액
                    ↓
3. 팝빌 API 호출하여 전자세금계산서 발행
                    ↓
4. 국세청 자동 신고 + 이메일 자동 교부
```

#### 위수탁 발행 가능

사인노트가 수탁자로서 각 협력업체(위탁자) 대신 세금계산서를 발행하는 **위수탁 발행**도 지원되므로, 협력업체가 직접 발행하지 않아도 됩니다.
