# Signnote 디자인 및 비즈니스 분석 보고서

## 1. 프로젝트 개요

**앱 이름:** Signnote (사인노트)
**도메인:** 아파트 사전점검/입주박람회 입주옵션품목(유상옵션) 계약 플랫폼
**플랫폼:** 모바일 앱 (iOS/Android) + 관리자 웹 (PC)
**언어:** 한국어

---

## 2. 사용자 유형 (4가지 역할)

### 2-1. 고객 (Customer) — 📱 모바일 앱
- 아파트 입주 예정자
- 박람회에서 입주옵션품목(유상옵션)을 브라우징하고 계약(구매)하는 사용자
- **진입 방식:** 참여 코드 입력으로 입장
- **주요 흐름:** 코드 입장 → 평형 선택 → 행사 목록 → 상품 상세 → 장바구니 → 계약금 결제 → 계약함 확인

### 2-2. 협력업체 (Vendor) — 📱 모바일 앱
- 입주옵션품목(유상옵션) 판매업체
- 자사 상품을 등록하고 계약 관리를 하는 사용자
- **진입 방식:** 참여 코드 입력으로 입장
- **주요 흐름:** 코드 입장 → 행사 목록 → 품목 관리(추가/수정) → 계약함(집계 + 계약 건 관리)

### 2-3. 주관사 (Organizer) — 💻 웹 (PC)
- 박람회 주관 업체
- 행사를 생성하고 참여 코드를 발급, 업체·상품을 관리하며 전체 매출을 확인
- **주요 흐름:** 행사 생성 → 참여 코드 발급 → 품목 관리 → 계약/매출 대시보드

### 2-4. 관리자 (Admin) — 💻 웹 (PC)
- 시스템 전체 관리자
- 주관사/업체/고객 관리, 정산 관리, 시스템 설정
- **주요 흐름:** 전체 행사 관리 → 사용자 관리 → 정산 관리 → 시스템 설정

---

## 3. 비즈니스 플로우

```
주관사(행사 생성/관리)
    ↓ 참여 코드 발급
협력업체(품목 등록/계약 관리) ←→ 고객(품목 선택 → 장바구니 → 계약금 결제)
    ↓
계약금 결제 → PG 정산 → 업체별 금액 분배 → 세금계산서 자동 발행
```

## 5. 공통 UI 컴포넌트

### 5-1. 네비게이션
- **상단 헤더:** 뒤로가기 화살표 + 제목 텍스트 (예: "창원 자이 사전 박람회")
- **상태 바:** 시간(9:41), 신호, 배터리 표시
- **하단 탭바:** 홈 | 장바구니 | 계약함 | 마이페이지 (4탭)

### 5-2. 버튼
- **주요 액션 버튼:** 파란색 (고객용) / 검정색 (업체·주관사용)
- **장바구니 버튼:** 빨간 뱃지로 수량 표시
- **보조 버튼:** 라인 스타일, "샘세보기", "취소 요청" 등

### 5-3. 카드 컴포넌트
- **행사 카드:** 커버 이미지 + 행사명 + D-day + 날짜
- **상품 카드:** 상품 이미지 + 상품명 + 가격(빨간색)
- **계약 카드:** 상품명 + 가격 내역 + 상태

### 5-4. 모달
- **평형 선택 모달:** 라디오 선택 + 완료 버튼
- **상품 상세 모달:** 이미지 + 정보 + 닫기(X) + 액션 버튼

### 5-5. 아이콘
- 홈(집), 장바구니(카트), 계약함(문서), 마이페이지(사람)
- 셰브론(화살표), Info(정보), 수정(연필)

## 8. 결제 및 정산 시스템

### 8-1. PG사 결제
- **결제 주체:** 고객이 계약금 결제
- **PG사 후보:** KG이니시스, 토스페이먼츠, NHN KCP
- **결제 수단:** 신용카드, 계좌이체 등

### 8-2. 지급대행 (정산 분배)
고객이 한 번에 여러 업체 상품을 결제하면, PG사의 **지급대행 서비스**를 통해 업체별로 자동 분배

```
고객이 500만원 결제 (줄눈 140만 + 나노코팅 70만 + ...)
    ↓
PG사가 대금 수령 (수수료 차감)
    ↓
사인노트 서버가 PG사에 정산 분배 데이터 전송
- A업체(줄눈): 140만원 → A업체 계좌
- B업체(나노코팅): 70만원 → B업체 계좌
- 사인노트 수수료: X만원 → 사인노트 계좌
    ↓
PG사가 각 업체 계좌로 자동 입금
```

### 8-3. 지급대행 서비스 PG사

| PG사 | 서비스명 | 특징 |
|------|---------|------|
| **KG이니시스** | 지급대행 서비스 | 하위 판매자 계좌 기반 자금 정산 대행 |
| **토스페이먼츠** | 서브몰 정산 | 플랫폼 내 하위 상점별 자동 정산 |
| **NHN KCP** | 지급대행 | 대표가맹점 방식 하위 가맹점 정산 |

### 8-4. 주의사항
- 전자지급결제대행업(PG업) 등록 필요 여부 확인 (PG사에 위탁 시 면제 가능)
- 보증보험 가입 필수
- 수수료: 신용카드 기준 약 3.2~3.5%

## 9. 세금계산서 자동 발행

### 9-1. 서비스 후보

| 서비스 | 특징 | 가격 |
|--------|------|------|
| **팝빌 (Popbill)** | 국세청 표준인증, Bulk API, Webhook, 14개+ SDK | 건당 과금 |
| **바로빌 (BaroBill)** | 정발행/역발행/위수탁발행 모두 지원 | 건당 과금 |

### 9-2. 자동화 시나리오

```
고객이 계약금 결제 완료
    ↓
사인노트 서버가 세금계산서 데이터 자동 생성
- 공급자: 해당 협력업체
- 공급받는자: 고객 (또는 사인노트)
- 금액: 해당 품목 금액
    ↓
팝빌 API 호출하여 전자세금계산서 발행
    ↓
국세청 자동 신고 + 이메일 자동 교부
```

### 9-3. 위수탁 발행
사인노트가 수탁자로서 각 협력업체(위탁자) 대신 세금계산서를 발행하는 **위수탁 발행** 가능 → 협력업체가 직접 발행하지 않아도 됨

## 12. 데이터 모델

### 12-1. 주요 엔티티
- **User (사용자):** 역할(고객/업체/주관사/관리자), 이름, 연락처, 사업자번호(업체)
- **Event (행사):** 행사명, 결제방식, 주관사, 세대수, 주거타입, 기간, 커버이미지, 참여코드
- **Product (상품/품목):** 상품명, 주거타입, 이미지, 설명, 가격, 소속 업체, 소속 행사
- **Cart (장바구니):** 사용자, 상품 목록
- **Contract (계약):** 고객, 상품, 가격(원가/할인/최종), 상태
- **Payment (결제):** 계약 ID, PG 거래번호, 결제금액, 상태
- **Settlement (정산):** 업체별 정산금액, 정산상태, 지급일
- **TaxInvoice (세금계산서):** 공급자, 공급받는자, 금액, 발행상태, 국세청 승인번호

### 12-2. 관계
- 주관사 1:N 행사
- 행사 N:M 업체
- 업체 1:N 상품
- 행사 1:N 상품
- 고객 1:1 장바구니
- 고객 1:N 계약
- 계약 1:1 결제
- 결제 1:N 정산 (업체별)
- 정산 1:1 세금계산서
