# Signnote 수수료 설계안 (초안)

> 작성일: 2026-03-16
> 상태: 계획 단계 (미구현)
> 결정 필요: 방식 선택, 비율 확정

---

## 1. 수수료 종류

Signnote 플랫폼에서 발생하는 수수료는 **3가지**:

| 수수료 | 설명 | 누가 정함 | 누가 부담 |
|--------|------|----------|----------|
| ① 주관사 수수료 | 주관사가 업체 매출에서 가져가는 몫 | 주관사 (품목별 설정) | 협력업체 |
| ② 플랫폼 수수료 | Signnote 서비스 사용료 | 관리자 (시스템 설정) | 아래 방식별 |
| ③ 카드 수수료 | PG사 카드 결제 수수료 | PG사 (토스/이니시스) | 아래 방식별 |

---

## 2. 현재 구조 (수수료 ①만 있음)

```
고객이 1,400,000원 계약, 계약금 30% = 420,000원 카드결제 시:

  총 계약금액:       1,400,000원
  ├─ 주관사 수수료 20%:  280,000원  → 주관사
  └─ 나머지:          1,120,000원  → 협력업체

  플랫폼 수익:  0원
  카드수수료:   고려 안 됨
```

---

## 3. 플랫폼 수수료 (② Signnote 서비스 사용료)

### 방식 A: 총액에서 먼저 차감 (추천)

```
  총 계약금액:        1,400,000원
  ├─ 플랫폼 수수료 1%:   -14,000원  → Signnote
  └─ 남은 금액:       1,386,000원
      ├─ 주관사 (20%):   277,200원  → 주관사
      └─ 협력업체:     1,108,800원  → 협력업체
```

- 장점: "총액의 1%"로 설명 간단, 양쪽 공평하게 간접 부담
- 단점: 주관사/업체 모두 수익 약간 줄어듦

### 방식 B: 양쪽 각각 부과

```
  총 계약금액:        1,400,000원
  ├─ 주관사 몫 (20%):   280,000원
  │   └─ 플랫폼 1%:      -2,800원  → Signnote
  │   └─ 실수령:        277,200원
  └─ 협력업체 몫:     1,120,000원
      └─ 플랫폼 1%:     -11,200원  → Signnote
      └─ 실수령:      1,108,800원
```

- 장점: 각자 자기 몫에서 부담, 명확
- 단점: "수수료에서 또 수수료" 느낌

### 방식 C: 협력업체에만 부과

```
  총 계약금액:        1,400,000원
  ├─ 주관사 (20%):     280,000원 (변동 없음)
  └─ 협력업체 몫:    1,120,000원
      └─ 플랫폼 1%:    -14,000원  → Signnote
      └─ 실수령:     1,106,000원
```

- 장점: 주관사 불만 없음, 업체가 판매 수익에서 부담
- 단점: 업체만 이중 수수료 (주관사 + 플랫폼)

## 4. 카드 수수료 (③ PG사 결제 수수료)

### 한국 카드수수료 현황

| PG사 | 일반 가맹점 | 영세 가맹점 |
|------|-----------|-----------|
| 토스페이먼츠 | 2.5~3.5% | 1.5~2.0% |
| KG이니시스 | 2.5~3.3% | 1.5~2.0% |

- **중요**: 한국 법률상 카드수수료를 고객에게 전가하는 것은 금지 (여신전문금융업법)
- 따라서 고객 결제 금액은 변동 없고, 수수료는 판매자(가맹점) 측에서 흡수

### Signnote에서 카드수수료가 발생하는 시점

고객이 **계약금**을 카드로 결제할 때 → PG사가 수수료 차감 후 정산

```
  고객 결제:     420,000원 (계약금)
  PG 카드수수료 3%: -12,600원
  PG 정산액:     407,400원  ← 실제 입금되는 금액
```

### 카드수수료 처리 방식

#### 방식 ①: 협력업체가 부담 (가장 일반적)

```
  계약금액:          1,400,000원
  계약금 (30%):        420,000원 (고객 결제)
  카드수수료 3%:       -12,600원
  ──────────────────────────────
  실 입금:             407,400원

  이 407,400원에서 주관사/플랫폼 수수료를 차감하고 업체에 지급
  → 카드수수료는 결과적으로 업체가 부담
```

장점: 배민/쿠팡/스마트스토어 등 대부분의 플랫폼이 이 방식
단점: 업체 부담이 큼 (주관사수수료 + 카드수수료 + 플랫폼수수료)

#### 방식 ②: 주관사와 업체 비율 분담

```
  카드수수료 12,600원을 주관사수수료 비율(20:80)로 분담
  ├─ 주관사 부담 (20%):  2,520원
  └─ 협력업체 부담 (80%): 10,080원
```

장점: 공평
단점: 계산 복잡, 주관사 반발 가능

#### 방식 ③: 플랫폼(Signnote)이 부담

```
  카드수수료 12,600원을 Signnote가 전액 부담
  → 플랫폼 수수료 수입(14,000원)에서 카드수수료(12,600원)를 빼면
  → 실제 플랫폼 수익 = 1,400원
```

장점: 업체/주관사 부담 없음 → 서비스 매력도 ↑
단점: 플랫폼 수익이 거의 없어짐 (수수료율 높여야 함)

## 5. 종합 시나리오 (전체 수수료 흐름)

### 추천 조합: 플랫폼 방식A + 카드 방식①

```
고객 계약: 1,400,000원 (계약금 420,000원 카드결제)

[1단계] PG 결제 정산
  고객 결제:            420,000원
  카드수수료 (3%):       -12,600원
  PG 입금액:            407,400원

[2단계] 수수료 계산 (총 계약금액 기준)
  총 계약금액:         1,400,000원
  플랫폼 수수료 (1%):    -14,000원  → Signnote
  배분 대상 금액:      1,386,000원
  주관사 수수료 (20%):   277,200원  → 주관사
  협력업체 몫:        1,108,800원  → 협력업체

[3단계] 실제 정산 (카드수수료 반영)
  ※ 계약금만 카드결제, 잔금은 별도

  계약금 420,000원 중:
  ├─ PG 입금:    407,400원
  ├─ 플랫폼 수수료 배분: 계약금 비율로 차감
  └─ 나머지를 주관사/업체에 정산

[정산 요약]
  총 계약금액:    1,400,000원
  ├─ Signnote:      14,000원  (플랫폼 1%)
  ├─ PG사:          12,600원  (카드 3%)
  ├─ 주관사:       277,200원  (20% of 남은 금액)
  └─ 협력업체:   1,096,200원  (나머지)

  합계: 14,000 + 12,600 + 277,200 + 1,096,200 = 1,400,000원 ✓
```

## 9. 결정이 필요한 사항

- [ ] 플랫폼 수수료 방식: A(총액차감) / B(양쪽부과) / C(업체만)
- [ ] 플랫폼 수수료율: 1% / 1.5% / 2%
- [ ] 카드수수료 부담: 협력업체 / 비율분담 / 플랫폼
- [ ] PG사 수수료율: 3% (확인 필요)
- [ ] 기존 계약에 소급 적용 여부
- [ ] 고객에게 수수료 표시 여부 (표시 안 함 추천)
